Theory & Discussion

A deep dive into polynomial interpolation, numerical stability, complexity analysis, and mathematical proofs.

Newton vs. Lagrange Interpolation

Both Newton's divided difference method and the Lagrange method yield the exact same unique interpolating polynomial for a given set of points. However, their algebraic structures are vastly different, which leads to different computational characteristics.

1. Incremental Updates

If we have a polynomial \(P_{n-1}(x)\) interpolating \(n\) points, and we wish to add a new point \((x_n, y_n)\):

  • Lagrange Method: Requires recalculating every cardinal basis polynomial \(L_i(x)\) from scratch. The previous work cannot be reused. This takes \(O(n^2)\) operations.
  • Newton Method: We only need to compute the new row of divided differences corresponding to \(x_n\). The existing coefficients \(a_0, \dots, a_{n-1}\) remain unchanged. We just add one new term to the polynomial: \[P_n(x) = P_{n-1}(x) + f[x_0, x_1, \dots, x_n]\prod_{i=0}^{n-1}(x-x_i)\] This incremental step takes only \(O(n)\) operations.

2. Computational Complexity

Operation Lagrange Newton
Table Setup None \(O(n^2)\)
Interpolation (Single Point) \(O(n^2)\) \(O(n)\) (using nested multiplication)
Adding a Data Point \(O(n^2)\) \(O(n)\)

Mathematical Derivation

We assume the interpolating polynomial has the following lower-triangular form: \[P_{n-1}(x) = c_0 + c_1(x-x_0) + c_2(x-x_0)(x-x_1) + \dots + c_{n-1}(x-x_0)\cdots(x-x_{n-2})\] To find the coefficients \(c_k\), we substitute the data points successively:

  • Substitute \(x = x_0\): \(P_{n-1}(x_0) = c_0 = y_0 \implies c_0 = f[x_0]\)
  • Substitute \(x = x_1\): \(P_{n-1}(x_1) = c_0 + c_1(x_1 - x_0) = y_1 \implies c_1 = \frac{y_1 - y_0}{x_1 - x_0} = f[x_0, x_1]\)
  • By induction, we find that the \(k\)-th coefficient is the \(k\)-th order divided difference: \[c_k = f[x_0, x_1, \dots, x_k]\]

Divided differences satisfy the recurrence relation: \[f[x_i, \dots, x_{i+k}] = \frac{f[x_{i+1}, \dots, x_{i+k}] - f[x_i, \dots, x_{i+k-1}]}{x_{i+k} - x_i}\] This allows us to arrange the calculations in an upper-triangular table.

Runge's Phenomenon & Interpolation Errors

It is tempting to assume that increasing the number of interpolation points \(n\) (and thus the degree of the polynomial) always improves accuracy. However, this is not always true.

Runge's Phenomenon

When interpolating functions with high-degree polynomials over equally spaced points, large oscillations can occur near the edges of the interval. A classic example is the Runge function: \[f(x) = \frac{1}{1 + 25x^2}\] Over the interval \([-1, 1]\), as \(n \to \infty\), the interpolation error near the boundaries grows without bound: \(\lim_{n\to\infty} \max |f(x) - P_n(x)| = \infty\).

Error Bound Formula

If \(f(x)\) has \(n\) continuous derivatives, the error of the interpolating polynomial \(P_{n-1}(x)\) is given by:

\[E(x) = f(x) - P_{n-1}(x) = \frac{f^{(n)}(\xi)}{n!} \prod_{i=0}^{n-1} (x - x_i)\]

where \(\xi\) is some unknown point in the interval containing all the nodes and \(x\).

Mitigating Interpolation Errors

  • Chebyshev Nodes: Instead of equally spaced points, choose points clustered near the boundaries (roots of Chebyshev polynomials). This minimizes the product term \(\prod (x-x_i)\) and eliminates Runge's oscillations.
  • Spline Interpolation: Instead of a single high-degree polynomial, fit low-degree polynomials (usually cubics) piecewise between adjacent points.
